# 项目落地计划

## 原则

1. 先完成详细设计，再继续落地代码。
2. 每一阶段都必须能运行、能验证，不做只存在于文档里的架构。
3. 不用硬编码例外堆功能，优先抽象共同结构。
4. AI 是系统的一部分，不只是外部工具。编排、提示词、输出协议和审阅流程都属于工程设计。

## 阶段一任务清单

### 1. 数据模型校正

目标：让代码中的数据结构与 [detailed_design.md](./detailed_design.md) 对齐。

任务：

1. 保留现有 `Provider`。
2. 在 `aiRoles` 中增加 `planner`，用于小说策划助手。
3. 增加 `archives`，保存背景、大纲、角色、场景、线索等项目档案。
4. 增加 `planning.messages` 和 `planning.roleplayDrafts`，保存策划对话和 AI 生成的扮演配置草案。
5. 保留角色卡基础字段。
6. 保留 `session.events`，记录每次 AI 调用和用户操作。

验收：

1. 老数据能被迁移或兼容读取。
2. 前端能显示所有小说、角色、提供商。

### 2. 后端 API

目标：完成本地 API 闭环。

任务：

1. 小说增删改查。
2. 策划 Agent 对话、运行控制和历史记录。
3. 策划 Agent 通过工具调用检索资料、沉淀档案、更新记忆和世界书。
4. 策划 Agent 根据当前档案生成扮演配置草案。
5. 采纳草案或人工修改配置。
6. 角色增删改查。
7. 提供商增删改查。
8. 模型查询和手动添加。
9. 扮演运行。
10. 小说改写。
11. 上下文包预览、记忆重建和 AI 记忆整理。
12. 正文草稿、采纳、废弃和编辑接口。

验收：

1. 所有接口返回统一 JSON。
2. 错误信息可读。
3. key 不完整回显。

### 3. 前端工作台

目标：完成可用的多页面界面。

任务：

1. 小说库。
2. 小说打开后的 AI 策划页。
3. 档案页。
4. 扮演配置页。
5. 行文页。
6. 提供商页。
7. 角色卡编辑。
8. Agent 模型配置。
9. 扮演记录和正文展示。
10. 记忆页。
11. 正文草稿、采纳、废弃和编辑控制。
12. 模型查询候选列表，用户手动选择添加。

验收：

1. 用户不需要改 JSON 文件即可完成配置。
2. 用户能通过 AI 策划对话生成档案。
3. 用户能由 AI 生成扮演配置并采纳。
4. 一轮扮演能从前端启动。
5. 改写结果能在前端看到。
6. 用户能审查上下文包，且草稿正文不会直接进入稳定记忆。

### 4. 提示词与输出协议

目标：让 AI 输出尽量结构化。

任务：

1. 导演提示词。
2. 主要角色提示词。
3. 次要角色群提示词。
4. 改写提示词。
5. 策划助手提示词。
6. 档案提取提示词。
7. 扮演配置生成提示词。
8. JSON 解析失败时保留原文并提示用户。

验收：

1. 正常模型能输出可解析结构。
2. 解析失败不会丢失内容。
3. 改写正文没有 JSON 痕迹。

### 5. 文档

目标：让项目可以继续维护。

任务：

1. 更新 README。
2. 记录运行方式。
3. 记录提供商配置示例。
4. 维护 downloads 文档。
5. 继续保持 `user_ideas.md` 只记录用户思路。

## 阶段一完成标准

1. `npm start` 能启动本地工作台。
2. 浏览器可以打开前端。
3. 至少能配置一个提供商。
4. 能通过策划对话生成档案。
5. 能由 AI 生成扮演配置。
6. 至少能添加或采纳一个主要角色和一个次要角色。
7. 能运行一轮扮演。
8. 能生成一段小说正文。
9. 正文草稿不会进入稳定记忆，采纳后进入正文库和 RAG 证据层，并由 AI 整理长期事实。
10. 文档说明清楚当前能力和限制。

## 阶段一完成情况

当前代码已经完成阶段一闭环：

1. `npm start` 可以启动本地工作台。
2. 小说库、AI 策划、档案、记忆、扮演配置、行文、提供商页面已落地。
3. 本地模拟提供商可以跑通策划、配置、扮演、改写、采纳正文和记忆整理。
4. 模型查询只展示候选，不直接写入提供商模型列表。
5. 记忆系统已接入实际 AI 调用，且前端可以审查每轮注入。
6. PNG 角色卡元数据导入、独立世界书编辑器和分策略记忆组包已落地。
7. 本地向量索引、embedding/rerank、导演写前定位、审查链和单角色重跑已落地。

阶段一之后仍需要继续强化的是“过程可控性”，不是继续堆更多配置字段。优先级应是扮演记录人工编辑、档案提取审核、审查结果差异化展示和正文版本分支。

## 暂不做

1. 不做云端账号系统。
2. 不做多人协作。
3. 不做复杂权限。
4. 不完整复刻酒馆所有世界书扩展字段。
5. 当前只做本地文件型向量索引，不接外部专用向量数据库服务。
6. 不做全自动长篇连载。

这些功能不是没价值，而是会分散第一阶段的工程焦点。
